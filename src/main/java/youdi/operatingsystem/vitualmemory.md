### Sawp技术

地址空间
是一块连续的内存块


碎片问题：
进程来回分配， 回收交换， 内存之间会产生很多缝隙

频繁切换问题：
    如果进程过多，内存较小，会频繁触发交换



设计目标：
    
    隔离：每个应用有自己的地址空间，互不影响、
    性能： 高频使用的数据保留在内存中，低频使用的数据持久化到磁盘中
    程序好写： 不用关心底层设施



### 虚拟化内存

    应用 -> 页(虚拟内存) -> 真实内存

每个应用都有高频使用的数据和低频使用的数据
按页进行缓存



虚拟内存
    页 -> 页表 -> frame

1. 通过虚拟地址计算page编号
2. 查页表，根据page编号，找到Frame编号
3. 将虚拟地址算成物理地址
